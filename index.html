Comprendre les commandes Git

1. git init → Initialiser Git

Tu dis à Git : "Surveille ce dossier" Résultat : un dossier caché .git est créé → Git commence à suivre les fichiers. Git init

2. (optionnel) Créer une branche nommée main

C’est ta "ligne principale" de travail/ git branch -M main

3. git remote add origin → Relier à GitHub

Tu dis à Git où envoyer ton code /git remote add origin https://github.com/TON-PROFIL/TON-DEPOT.git origin veut dire: dépot de github

4. git add . → Préparer les fichiers

Tu dis : "je veux inclure ces fichiers dans la prochaine sauvegarde"/ git add . . = tous les fichiers modifiés ou nouveaux.

5. git commit -m "message" → Sauvegarder localement

Tu fais une capture de ton projet, avec un commentaire git commit -m "Ajout de la page d’accueil" Ce commit est enregistré uniquement sur ton ordinateur pour l’instant.

6. git push -u origin main → Envoyer sur GitHub

Tu dis : "envoie ce commit sur la branche main de mon GitHub"/ git push -u origin main Maintenant ton site/code est en ligne sur GitHub.

# 1. Initialise le dossier git init
# 2. Crée la branche principale git branch -M main
# 3. Relie à ton dépôt GitHub git remote add origin https://github.com/TON-PROFIL/TON-DEPOT.git
# 4. Ajoute tous les fichiers git add .
# 5. Fais un premier commit git commit -m "Premier commit"
# 6. Envoie tout sur GitHub git push -u origin main

A chaque modification
git add . git commit -m "Description de ce que tu as changé" git push
